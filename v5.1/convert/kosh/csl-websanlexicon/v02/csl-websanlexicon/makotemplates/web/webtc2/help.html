Help for Advanced Search for ${dictname}

This program does partial word searches for Sanskrit head words or other text words in the dictionary. For the basic usage, type a word in the text input box next to 'Sanskrit word:' or 'Text word:', set the search paramenters, and the maximum and output parameters, and click the button. A message in the left display pane indicates the program is working. When the request is finished, output appears in the left display pane.

Instead of clicking the button, one can press the key while the cursor is in the text input box. %if False: (NOTE: Currently, this is known to work properly only when using the Firefox or IE7 browsers with the Windows XP operating system. It is known to work improperly for the Safari browser on the Macintosh; however, the 'Search' button seems to work ok in this case.) %endif

The data in the left pane presents the underlying Sanskrit words where matches occur; if you are searching for a text word, the matching word appears in parentheses next to the Sanskrit word. The Sanskrit word is clickable. If clicked, the full record for the word appears in the right-hand pane. All Sanskrit words appear according to the setting of the 'output' parameter.

The Text words include words which are not coded as Sanskrit words; generally, the words are coded as Sanskrit if they appear in Devanagari in the printed text. The search is not case-sensitive. %if False: all non-Sanskrit words. For the latter, the spelling to use is the spelling of the word as it appears, but without the various additional marks. For example a search for vermogen matches Vermögen. Similarly, a sarch for siddh matches the literary citation abbreviation SIDDH.K. %endif

A search for a Sanskrit word includes both the headwords, and the Sanskrit words appearing in Devanagari in the scan edition of the text.

Various controls allow some degree of tuning of the search process and of the output format.

  • exact/prefix/suffix/substring (Sanskrit or textword). This determines where within a possibly longer word the match occurs.
  • HK/SLP1/ITRANS (Sanskrit word): the transliteration scheme in which you input the Sanskrit word.
  • Maximum: this number determines the maximum number of matches per search. Bigger numbers will take longer, of course.
  • output: This allows a choice of how to represent the Sanskrit words in the output, in each pane.
  • button: This button is available after a search in which the maximum number of matches did not exhaust the dictionary. In other words, there may be more matches. Clicking the button continues the search, from the point at which the prior search ended.

working... workaround
Some users have noticed that after a data request is made, 'working...' appears, and then the usual data display never appears. We are aware of this problem, but unsure of how to prohibit it. It seems to happen more frequently for some users than for others. Experience has shown that one of several similar actions can usually get the user past this problem when it occurs:

  • Press again
  • OR, Re-enter the search parameters, then press again.
  • OR, Press the 'reload' button for your web browser, then re-enter the search parameters, then press again.
